# Contributing & Security Review Policy

Because extensions run **in-process with no sandbox**, trusting this repo means trusting our signing
key with full device access for every APK it signs. Review is not optional — it is the security model.

## Hard rules (a PR that violates any is rejected)

1. **Every network call is reviewed.** `GET`/`POST`/`client.newCall(...)` may only target a domain
   that is a **literal** in the source or the user-overridable `baseUrl`. No request URL may be
   assembled from decoded, decrypted, or runtime-fetched strings.
2. **No obfuscation.** No base64/hex/xor "decode-then-eval", no `Class.forName`/reflection inside an
   extension, no `DexClassLoader`/dynamic dex, no downloading code at runtime. If a reviewer can't
   read what it does, it does not merge.
3. **Least privilege.** No Android permissions. No `<activity>`/services unless a `UrlActivity` for
   deep links is genuinely required. Use the host `NetworkHelper` client — never open raw sockets.
4. **Build-from-source only.** Never add a prebuilt APK. The `repo` branch is generated by CI from
   reviewed `main` source.
5. **Bump `extVersionCode` on every change** so users receive the update. Bump `versionId` only when
   a site's URL scheme changes incompatibly (it creates a fresh source id; see resilience notes).

## Review checklist (paste into each PR review)

- [ ] All request URLs trace to a literal domain or `baseUrl` (diff the request set).
- [ ] No reflection / dynamic loading / encoded-string execution.
- [ ] DTOs use defaults + nullable types (won't break on an added/missing API field).
- [ ] Uses `setUrlWithoutDomain(...)` for stored manga/chapter URLs (domain-change safe).
- [ ] Selectors centralized (theme override, not duplicated logic).
- [ ] `extName/extClass/extVersionCode/isNsfw` correct; `versionName` lands in the loader range.
- [ ] No new permissions, activities, or services.

## Style

- Kotlin official style; prefer JSON APIs (`HttpSource` + `parseAs`) over HTML scraping.
- For HTML sites, extend a `lib-multisrc` theme; add a new theme only when ≥2 sites share a CMS.
- `ParsedHttpSource` is acceptable for simple HTML but is deprecated upstream — prefer `HttpSource`.
